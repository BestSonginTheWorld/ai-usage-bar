AIUsageMenuBar 설치 안내

필수 환경
- macOS 13 이상
- tmux 설치
- claude CLI 설치
- codex CLI 설치

배포 파일
- AIUsageMenuBar.app
- install.sh
- uninstall.sh
- ai_usage_collector.py
- config.example.json

설치 방법
1. GitHub Releases에서 최신 `AIUsageMenuBar-<version>.zip` 파일을 다운로드합니다.
2. 압축을 풉니다.
3. 터미널을 열고, 압축을 푼 폴더로 이동합니다.
4. 아래 명령을 실행합니다.

   chmod +x install.sh
   ./install.sh --use-prebuilt

설치 후 생성되는 경로
- 앱: ~/Applications/AIUsageMenuBar.app
- 설정/로그/캐시: ~/Library/Application Support/AIUsageMenuBar/
- 자동 시작 등록: ~/Library/LaunchAgents/local.ai-usage-menubar-app.plist

처음 실행 시 macOS 경고가 뜰 수 있음
- 이 앱은 현재 Apple notarization이 되어 있지 않습니다.
- 다른 Mac에서는 처음 실행할 때 "Apple이 악성 소프트웨어 여부를 확인할 수 없어 열 수 없습니다" 또는 비슷한 경고가 뜰 수 있습니다.
- 이 경우 Finder에서 AIUsageMenuBar.app를 우클릭한 뒤 "열기"를 선택하거나, 시스템 설정 > 개인정보 보호 및 보안에서 "그래도 열기"를 선택하면 됩니다.

필수 도구가 없을 때
- install.sh 자체는 실행됩니다.
- 하지만 사용량 수집은 동작하지 않습니다.
- 앱 메뉴에서 해당 provider가 unavailable 또는 not installed 상태로 표시될 수 있습니다.
- 즉, tmux/claude/codex가 없다고 macOS 보안 경고창이 뜨는 것은 아닙니다.

권장 설치 예시
- Homebrew 사용 시:
  brew install tmux

설정 파일
- 기본 설정 파일은 다음 경로에 생성됩니다.
  ~/Library/Application Support/AIUsageMenuBar/config.json

삭제 방법
1. 터미널에서 배포 폴더로 이동합니다.
2. 아래 명령을 실행합니다.

   chmod +x uninstall.sh
   ./uninstall.sh

주의
- uninstall.sh는 앱과 LaunchAgent만 제거합니다.
- 로그, 캐시, 설정이 들어 있는 ~/Library/Application Support/AIUsageMenuBar/ 는 자동 삭제하지 않습니다.
